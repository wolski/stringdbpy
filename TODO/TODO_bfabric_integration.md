# B-Fabric Integration Plan: STRING-GSEA Docker

## Goal

Replace A373_STRING_GSEA in slurmworker with a Docker-based integration using the `wolski/string-gsea` image. This supersedes both A373 (old Python-only GSEA, no reports) and A383 (fragile Snakemake + python_env with R/Quarto deps on the node).

## Architecture

```
dispatch.py          -->  params.yml (workunit definition)
                          inputs/{sample}.zip (input resources)

process.py           -->  reads params.yml, extracts parameters
                          calls string_gsea_docker.sh with args
                          string_gsea_docker.sh runs Docker container
                          container runs string_gsea_workflow
                          outputs.yml written by container
```

### params.yml Structure (generated by dispatch)

```yaml
execution:
  raw_parameters:
    application_version: '0.1'
    1|FDR_threshold: '0.05'         # optional, maps to --fdr
    2|Filtering: 'pep_2_no_imputed' # maps to --which
registration:
  workunit_id: 322093
  workunit_name: STRING_GSEAv4
  container_id: 3000
  application_id: 373
  application_name: STRING_GSEA
  ...
```

### Parameter Mapping

| params.yml key | Docker CLI arg | Default |
|----------------|---------------|---------|
| `registration.workunit_id` | positional `workunit_id` | required |
| `execution.raw_parameters["2\|Filtering"]` | `--which` | `pep_2_no_imputed` |
| `execution.raw_parameters["1\|FDR_threshold"]` | `--fdr` | `0.25` (from config.toml) |

### 2. `dispatch.py`

Same as A383 — filter `.zip` resources, place in `inputs/`, serialize workunit definition as `params.yml`.

```python
import polars as pl
from bfabric_app_runner.dispatch.dispatch_resource_flow import ResourceDispatcherCLI

def dispatch_strategy(resources_df, workunit_definition):
    resources_df = resources_df.filter(pl.col("filename").str.ends_with(".zip"))
    resources_df = resources_df.with_columns(filename=pl.lit("inputs/") + pl.col("filename"))
    extra_inputs = [
        {
            "type": "static_yaml",
            "filename": "params.yml",
            "data": workunit_definition.model_dump(mode="json"),
        }
    ]
    return resources_df, extra_inputs

if __name__ == "__main__":
    ResourceDispatcherCLI(dispatch_strategy).run()
```

### 3. `process.py`

Python script that reads `params.yml`, extracts parameters, and calls `string_gsea_docker.sh`.

```python
"""Process script for A373 STRING-GSEA Docker integration.

Reads params.yml from the task directory, extracts workunit parameters,
and invokes string_gsea_docker.sh to run the analysis in Docker.
"""

import subprocess
import sys
from pathlib import Path

import yaml


def main():
    task_dir = Path(sys.argv[1])

    # Read params.yml
    with open(task_dir / "params.yml") as f:
        params = yaml.safe_load(f)

    workunit_id = str(params["registration"]["workunit_id"])
    raw = params["execution"]["raw_parameters"]
    filtering = raw.get("2|Filtering", "pep_2_no_imputed")
    fdr = raw.get("1|FDR_threshold")
    app_version = raw.get("application_version", "latest")

    # Find input zip
    zips = list((task_dir / "inputs").glob("*.zip"))
    if not zips:
        print("ERROR: No zip file found in inputs/", file=sys.stderr)
        sys.exit(1)
    zip_file = str(zips[0].relative_to(task_dir))

    # Build command: call string_gsea_docker.sh
    script_dir = Path(__file__).parent
    docker_sh = script_dir / "string_gsea_docker.sh"

    cmd = [
        str(docker_sh),
        "--image-version", app_version,
        "--",
        zip_file, workunit_id, ".",
        "--which", filtering,
        "--cores", "1",
    ]
    if fdr:
        cmd.extend(["--fdr", str(fdr)])

    # Run from task_dir so Docker mounts the right cwd
    result = subprocess.run(cmd, cwd=task_dir)

    # Verify outputs.yml was created
    if not (task_dir / "outputs.yml").exists():
        print("ERROR: outputs.yml not created by container", file=sys.stderr)
        sys.exit(1)

    sys.exit(result.returncode)


if __name__ == "__main__":
    main()
```

## GitHub Action: Docker Build + Push on Tag

New workflow at `.github/workflows/docker_publish.yml`. Only runs when a tag is pushed, and only after CI checks pass.

```yaml
name: Docker Publish

on:
  push:
    tags: ["v*"]

jobs:
  # Gate: run existing CI checks first
  quality:
    uses: ./.github/workflows/code_quality.yml
  tests:
    uses: ./.github/workflows/run_unit_tests.yml

  docker:
    name: Build and Push Docker Image
    needs: [quality, tests]
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write
    steps:
      - uses: actions/checkout@v4

      - name: Docker meta
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: |
            ghcr.io/wolski/string-gsea
          tags: |
            type=semver,pattern={{version}}
            type=semver,pattern={{major}}.{{minor}}
            type=raw,value=latest

      - uses: docker/setup-qemu-action@v3
      - uses: docker/setup-buildx-action@v3

      - name: Login to GHCR
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Build and push
        uses: docker/build-push-action@v6
        with:
          context: .
          file: docker/Dockerfile
          platforms: linux/amd64,linux/arm64
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
          cache-from: type=gha
          cache-to: type=gha,mode=max
```

This pushes to **GitHub Container Registry** (`ghcr.io/wolski/string-gsea`). Free for public repos, no Docker Hub rate limits.

The `string_gsea_docker.sh` `IMAGE_REPO` default should be updated to `ghcr.io/wolski/string-gsea`.
